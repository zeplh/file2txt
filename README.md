# 文本聚合打包器 · 为AI准备上下文

一个纯静态 HTML 工具，可以将多个文本文件（包括 ZIP 内的文件）合并成一个 TXT 或 Markdown 文件，方便交给 AI 分析。  
**所有处理均在浏览器本地完成，不会上传任何文件。**

[在线使用](https://你的用户名.github.io/仓库名/) | 无需安装，开箱即用

## 功能特性

- 📁 **支持多种文件格式**：自动尝试将任何文件作为文本读取（TXT、JSON、HTML、CSS、JS、代码文件等）。
- 🗜️ **解压 ZIP 包**：自动解析 ZIP 压缩包内的文件，递归提取其中的文本文件。
- ✏️ **修改文件路径**：可以为每个文件自定义虚拟路径（便于 AI 理解目录结构）。
- 🔁 **重试失败文件**：部分文件可能因编码问题读取失败，可一键重试或手动加入列表重新解析。
- 🎨 **自动深色模式**：跟随系统配色，日间/夜间自动切换。
- 📱 **移动端适配**：按钮和布局针对手机竖屏做了优化。
- 📤 **输出 TXT / Markdown**：可选择纯文本或 Markdown 格式，失败文件会保留路径和错误提示，AI 可按需向用户索取原始文件。

## 使用场景

- 将多个代码文件、配置文件、日志打包成一个文件，交给 ChatGPT / Claude 等 AI 分析。
- 整理项目文档、配置文件，快速生成上下文。
- 解压 ZIP 并合并其中的文本信息，便于一次性阅读。

## 快速开始

1. 打开工具页面（本地双击 `index.html` 或通过 GitHub Pages 访问）。
2. 点击 **添加文件** 或 **添加文件夹** 选择需要合并的文本文件。
3. 也可以点击 **添加ZIP包** 上传压缩文件，工具会自动解压并提取其中的文本文件。
4. 在列表中可修改每个文件的虚拟路径，或移除不需要的文件。
5. 选择输出格式（`.txt` 或 `.md`）。
6. 点击 **打包并保存**，浏览器将自动下载合并后的文件。

## 注意事项

- 工具依赖 [JSZip](https://stuk.github.io/jszip/) CDN，需要联网才能解析 ZIP 文件。
- 对于二进制文件（如图片、可执行文件），工具会尝试读取为文本并**可能显示乱码**，建议只选择纯文本格式的文件。
- 读取失败的文件不会出现在最终输出的内容中，但会保留其路径和失败原因，AI 可根据提示决定是否向用户索要原始文件。
- 所有数据均在您自己的浏览器中处理，不会上传到任何服务器，请放心使用。

## 部署到 GitHub Pages

您可以将本仓库直接部署到 GitHub Pages：

1. Fork 或下载本仓库。
2. 确保 `index.html` 在根目录。
3. 进入仓库 **Settings → Pages**，选择分支 `main` 和文件夹 `/ (root)`。
4. 访问 `https://你的用户名.github.io/仓库名/` 即可使用。

## 本地运行

直接双击 `index.html` 文件，或在本地用 `python -m http.server` 启动一个简单服务器，浏览器打开对应地址。
